## 1. 人脸与版权

### 上传人脸照片被审核拦截

**❌ 症状：** 参考照片在生成开始前就被拒或被标记。

**🔍 原因：** AI 视频平台的人脸审核越来越严，直接上传正脸照拒绝率最高。

**✅ 修复：** 按强度递进 ——
1. 多换几张照片试试。
2. 把照片丢给豆包，配合**商业人像描述模板**生成一版"写实风格的彩色
   素描画"，再上传那一版。
3. 把角色设计成**完全不露正脸**：头盔、面罩、机器人、背身镜头、戴大
   墨镜的人偶。（《丧尸清道夫》主角是机器人，真正的原因就在这。）

→ [faq.zh.md — 人脸 / 版权 / 违规](./faq.zh.md)

---

### 提示词被判版权 / IP 拦截

**❌ 症状：** 含假面骑士、高达、环太平洋、钢铁侠、卡莎等名字的提示词被
Seedance 2.0 拦下。

**🔍 原因：** Seedance 的版权过滤很激进，会拦真实的影视 / 角色名。

**✅ 修复：** 去掉名字，把**设计语言**留成描述。

```diff
- 钢铁侠红金战甲，掌心能量炮
+ 原子朋克复古未来感红金战衣，掌心能量爆发
- Black Sun 风格，硬核特摄
+ 暗黑战损质感，硬核特摄
```

也可以：删掉几个字符或标点而不改变含义，或换近义词描述同一种外观。

→ [faq.zh.md — 人脸 / 版权 / 违规](./faq.zh.md) ·
[prompts/README.md](./prompts/README.md)

### 人脸 / 身份在镜头间漂移

**❌ 症状：** 角色脸型或表情一条一条地变，跨剪辑点认不出是同一个人。

**🔍 原因：** 段 2 锁得不够死。不显式钉住五官，AI 就会漂。

**✅ 修复：** 在**段 2（人物与基础设定）**里把脸锁死：

```text
人脸：参考上传照片。五官、脸型、发型 100% 保留。不要美化。
      保留[伤口 / 疤痕 / 瑕疵]。
```

变身镜头要锁得更狠 —— 变身前后骨骼结构一致：

```text
变身前后保持完全相同的骨骼结构、下颌几何与面部五官，100% 一致。
```

每个关键视觉元素（脸、表情、关键道具）各给**一张专属参考图**，
避免 AI 把它们混在一起。

→ [methodology.zh.md — 段 2 人物与基础设定](./methodology.zh.md)

## 2. 真实感与五毛特效感

### 输出像廉价塑料 CG，不像实拍

**❌ 症状：** 脸和表面发亮、发假、像游戏渲染 CG，不像电影。

**🔍 原因：** 两个常见祸根 ——（1）过度美颜 / 滤镜的参考照片本身就假，
（2）参考图画质低、没有真实照片或 3D 渲染的质感。

**✅ 修复：** 用清晰、大尺寸、**无滤镜**的头像 —— 最好素颜（哪部电影会给
演员开美颜？）。只在素材本身是高细节写实图或 3D 渲染时才垫图。动态镜头
（变身、打斗）干脆不垫图，让 AI 从**纯文案**自由发挥。

→ [faq.zh.md — 画质 / 质感 / 真实感](./faq.zh.md)

---

### 参考图把它的画风带进了输出

**❌ 症状：** 输出沾上了参考图的**画风**（二次元、手绘、CG 渲染），而不只
是它的设计。

**🔍 原因：** AI 会把参考图的质感和画风信息绑在一起 —— 参考画质差，多余
的美学就泄漏进结果。

**✅ 修复：** 除非素材本身是真写实或高质量 3D 渲染，否则不要垫图。动态镜头
**干脆不垫图** —— Mx-Shell 的假面骑士战衣全部出自文案，这也让每次输出
都独一无二。参考图只用于角色 / 场景**连贯性**需求。

→ [methodology.zh.md — 原则 #1（不要给 AI 参考图）](./methodology.zh.md) ·
[faq.zh.md — 垫图 vs 纯文案](./faq.zh.md)

---

### 画面僵硬、像游戏 CG —— 没有"临场感"

**❌ 症状：** 镜头技术上没问题，但发假、机位锁死，像游戏过场而不是手持
电影瞬间。

**🔍 原因：** 缺少"呼吸感"手持浮动，也没有显式的真实感锚点。

**✅ 修复：** 在**段 1**加真实感锚点（`电影感 | 超真实 | 无游戏 CG 感`），
并在**段 4**里永远带上呼吸感那一行：

```text
呼吸感：手持镜头。全程保持极其轻微、如呼吸般的镜头浮动，增强临场感。
```

指明真实的相机 + 镜头，给 AI 一个具体的视觉锚点 —— 比如史诗镜头用
IMAX 电影摄影机 + Panavision C 系列（35mm，f/4），硬核科幻用
Sony Venice + Canon K-35。

→ [methodology.zh.md — 段 3 与段 4](./methodology.zh.md)

### 某个特定运镜执行不出来

**❌ 症状：** 焦段不对、角度不对，或者环绕方向反了。

**🔍 原因：** "电影感""有张力"这类含糊描述没有精度。AI 需要明确的相机型号、
焦段和运动词汇。

**✅ 修复：** 在**段 4**写精确 —— 相机 + 镜头、景别、角度、运动方式、节奏：

```diff
- 围绕主角来一段电影感、有张力的运镜
+ IMAX + Panavision C 35mm。中近景，低角度偏左 30°。
+ 顺时针环绕，极慢，带 0.1 秒反应式微抖。
```

可调用的词汇 —— 景别（远 / 中 / 近）、角度（低 / 高 / 平 / 过肩）、
运动（推 / 环绕 / 跟 / FPV）、节奏（极慢 / 平稳 / 突发 / 反应式）。

→ [methodology.zh.md — 段 4 运镜规则](./methodology.zh.md) ·
[faq.zh.md — 镜头与构图](./faq.zh.md)

---

### 后期调色就崩（断层、噪点）

**❌ 症状：** 调色稍微一推就出现断层和噪点。

**🔍 原因：** AI 生成视频**色彩位深低** —— 调色空间远不如实拍素材。

**✅ 修复：** 把色调**前置锁定**在出图 / 提示词阶段（**段 3，色彩与影调**），
并让每个镜头这一段保持一致。大的美学选择在提示词里定（好莱坞橙青、灰蓝、
复古暖橙 + 海盐蓝……）。CapCut 里只做小幅过渡 —— 别想在后期**修**色，
去提示词里修。

→ [faq.zh.md — 时长 / 长视频 / 剪辑](./faq.zh.md)

## 3. 运动与时长

### 一镜到底超过约 15 秒就崩

**❌ 症状：** 变身或单镜头跑太长，序列中段画质崩坏。

**🔍 原因：** Seedance 很难撑住远超 15s 的一镜生成。甜区是按场景类型
5–15s。

**✅ 修复：** 变身**每镜 ≤ 15s**。更长序列（>15s 打斗）先生成头 15s，再用
小云雀网页端的**"续写这段视频"**接下去。这种分段法比硬来一条 30s+ 的更稳。
Mx-Shell 的卡莎就是 15s 版本 + 单独一段 5s 延长。

→ [faq.zh.md — 时长 / 长视频 / 剪辑](./faq.zh.md) ·
[prompts/kaisa-transformation.md](./prompts/kaisa-transformation.md)

---

### 提示词超字数，提交不了

**❌ 症状：** 又长又细的多镜头提示词撞到字数上限，发不进 Seedance 2.0。

**🔍 原因：** 细分镜和多镜脚本会超过手机端的字数限制。

**✅ 修复：** 从手机 App 切到**小云雀网页端** —— 它字数上限更高。还是太长？
狠狠裁：删冗余形容词、合并相关短句、缩写场景行。（Mx-Shell 的环太平洋
提示词就明确标注需要网页端。）

→ [faq.zh.md — 字数 / 抽卡 / 复刻](./faq.zh.md) ·
[prompts/pacific-rim-gundam.md](./prompts/pacific-rim-gundam.md)

---

### 两个片段接不上 —— 像两条视频

**❌ 症状：** 把两段剪一起，看着像两个独立镜头，不像一个连续动作。

**🔍 原因：** 缺少运动方向连贯性。观众读不出一个动作怎么流进下一个。

**✅ 修复：** 靠**运动方向连贯性**衔接，而不是靠对齐静帧。《丧尸清道夫》的
例子：机器人扔出炸弹炸飞尸群、自己被推出画面一侧；下一镜，他**从同一侧
进画**。设计镜头时刻意安排出 / 入画的边，构建合逻辑的空间流动。

→ [faq.zh.md — 时长 / 长视频 / 剪辑](./faq.zh.md)

---

### 好镜头出了一次，再也复刻不出来

**❌ 症状：** 一条打斗或武器充能镜头一次出得很漂亮，之后每次重抽都复刻不出
那个运镜。

**🔍 原因：** AI 生成本质非确定。Mx-Shell 的《火焰恶魔》运镜是个意外 ——
AI 其实没照指令走，他自己也复刻不出来。

**✅ 修复：** 把随机当成流程的一部分接受。把一条提示词当成一张**抽卡券**，
而不是确定性配方 —— 把功夫下在重抽的**挑选**上，而不是提示词的打磨上。
Mx-Shell 的典型次数：高难度镜头 20+ 次，简单镜头 2–3 次。《丧尸清道夫》
烧了约 400 张图 + 200+ 条视频才落地约 40 条成片。**精挑输出，别追求
提示词完美。**

→ [faq.zh.md — 字数 / 抽卡 / 复刻](./faq.zh.md)

## 4. 音频

### AI 自己加了你没要的背景音乐

**❌ 症状：** 你只想要现场音，模型却给片段配了非现场配乐。

**🔍 原因：** 不另行说明时，AI 视频模型默认会加音乐。

**✅ 修复：** 永远在**段 3（氛围与画质）**里写明：

```text
声音：无配乐，仅保留现场音。
```

Mx-Shell 在每条提示词里都用这个公式（丧尸清道夫、假面骑士、卡莎、
合金装备充能打斗、环太平洋、赛博江湖）。

→ [methodology.zh.md — 段 3 氛围与画质](./methodology.zh.md)

## 5. 结尾

### 结尾堆特效，看着很廉价

**❌ 症状：** 结尾叠爆炸、光爆、粒子特效，压垮了画面，读起来很业余。

**🔍 原因：** 高潮处特效过载就是过度堆砌。Mx-Shell 刻意回避这一点。

**✅ 修复：** 练习克制 —— **安静收尾**。不要台词、不要爆炸、不要刺眼强光。
就让角色停在那个瞬间：站着、呼吸、在烟尘里，配一个远处的元素（一颗划过
天际的流星）。克制的结尾比堆满特效的更高级。

> 没有台词。没有爆炸。没有光芒万丈。只有身穿不完整战衣的人站在原地。
> —— Mx-Shell《方法论》原则 #3
